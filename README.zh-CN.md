# discord-author-exporter

中文说明 | **[English](./README.md)**

## 工作原理

扩展利用 Discord 网页端自带的搜索接口：

```
GET /api/v9/guilds/{guildId}/messages/search?author_id={用户ID}
```

- 拦截你**登录后**浏览器自身发出的请求，自动获取所需的授权令牌（仅保存在本地内存/存储，不上传任何地方）。
- 用 snowflake 游标(`max_id`)翻页抓取全部结果，不受 offset 5000 上限限制。
- 自动处理限速 (429) 与搜索索引未就绪 (202) 的等待重试。
- 网络中断时自动退避重试，并把已抓取的进度实时存档到浏览器本地 IndexedDB，可断点续传。
- 支持按用户名或用户 ID 查找目标；也可以直接复用 Discord 原生搜索栏刚发出的那条搜索请求，参数最可靠。

## 安装

1. 打开 Chrome，地址栏输入 `chrome://extensions/`。
2. 右上角打开「开发者模式」。
3. 点击「加载已解压的扩展程序」，选择本项目文件夹。
4. 扩展出现在工具栏后即可使用。

## 使用步骤

1. 在浏览器里打开 `https://discord.com/app` 并**登录**，进入目标服务器的任意一个频道。
2. 点击工具栏上的扩展图标，页面右上角会弹出一个**悬浮窗**（可拖动标题栏移动、可最小化/关闭，再点图标可切换显隐）。
3. 选择查找方式：
   - **按用户名**：先在页面上点开该用户的资料卡或看到其发言，让扩展捕获到他的信息，再输入用户名。
   - **按用户 ID**：开发者模式下右键头像「复制用户 ID」，粘贴进去，最稳妥。
   - 或者：先在 **Discord 自己的搜索栏**里搜一次「来自: 某人」，再点悬浮窗里的「使用 Discord 搜索框最近的用户」按钮，直接复用那条已验证有效的搜索参数。
4. 选择搜索范围（整个服务器 / 仅当前频道）和导出格式，需要的话勾选「仅保留含文本的消息」。
5. 点「开始导出」，等待进度完成，文件会自动下载。抓取过程中可随时点「取消」。

如果中途断网或关闭页面，进度不会丢——用相同的用户和范围再次点击「开始导出」时会自动从断点续传。

## 导出格式

| 格式 | 说明 |
|------|------|
| JSON | 含完整原始字段，适合二次处理 |
| CSV  | 时间 / 作者 / 内容 / 附件，可用 Excel 打开 |
| TXT  | 纯文本，便于阅读 |
| HTML | 仿 Discord 样式的网页，便于浏览 |

## 注意事项与限制

- 只能导出你**有权限查看**的频道内容；看不到的频道接口也不会返回。
- 按用户名查找依赖已被扩展捕获到的信息，找不到时请改用用户 ID 或复用原生搜索。
- 抓取速度受官方限速约束，消息量大时请耐心等待。
